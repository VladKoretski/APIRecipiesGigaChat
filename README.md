# Список покупок по блюду

Приложение для генерации списка покупок и шагов приготовления на основе названия блюда через LLM API (GigaChat).

## Функционал

- Ввод названия блюда (текст)
- Выбор количества персон: 1 / 2 / 4 / 6
- Выбор формата: «Список продуктов» / «Список + шаги»
- Валидация ввода (пустой/слишком длинный текст)
- Обработка ошибок API без вывода traceback
- Понятные сообщения пользователю

## Быстрый запуск

### 1. Клонирование и установка
```bash
# Клонировать репозиторий
git clone <ВАША_ССЫЛКА>
cd shopping_list_app

# Создать виртуальное окружение
python -m venv venv

# Активировать (Windows/PowerShell)
venv\Scripts\activate

# Установить зависимости
pip install streamlit requests python-dotenv
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корне проекта:
```env
GIGACHAT_CLIENT_ID=ваш_client_id
GIGACHAT_CLIENT_SECRET=ваш_client_secret
```
> Ключи получите в [личном кабинете GigaChat](https://developers.sber.ru/portal/products/gigachat)

### 3. Запуск приложения
```bash
streamlit run streamlit_app.py
```
Приложение откроется в браузере: `http://localhost:8501`

## Тест-сценарии проверки

### Сценарий 1: Нормальный ввод
**Действия:**
1. В поле «Что приготовить?» введите: `Паста Карбонара`
2. Выберите: `2` человека, формат `Список продуктов`
3. Нажмите «Сгенерировать список покупок»

**Ожидаемый результат:**  
Появляется список ингредиентов (макароны, бекон, яйца, сыр и т.д.), каждый с новой строки.
  
### Сценарий 2: Пустой ввод
**Действия:**
1. Оставьте поле «Что приготовить?» пустым
2. Нажмите «Сгенерировать список покупок»

**Ожидаемый результат:**  
Жёлтое предупреждение: `Напишите, что хотите приготовить.`  
Запрос к API **не отправляется**.
  
### Сценарий 3: Слишком длинный ввод
**Действия:**
1. Вставьте текст длиннее 200 символов (например, повторите "Тестовое блюдо " 15 раз)
2. Нажмите «Сгенерировать список покупок»

**Ожидаемый результат:**  
Жёлтое предупреждение: `Слишком длинный текст — сократите или разбейте на части.`  
Запрос к API **не отправляется**.

## Воспроизведение ошибки API

### Ошибка авторизации (401/403)
**Как воспроизвести:**
1. Откройте файл `.env`
2. Измените `GIGACHAT_CLIENT_ID` на неверное значение (например, добавьте символ в конец)
3. Сохраните файл и перезапустите приложение
4. Попробуйте сгенерировать список

**Ожидаемый результат:**  
Красное сообщение: `Не настроен доступ к сервису.`  
Traceback в интерфейсе **отсутствует**.

### Ошибка сети/таймаут
**Как воспроизвести:**
1. Отключите интернет или заблокируйте домен `gigachat.devices.sberbank.ru` в hosts-файле
2. Попробуйте сгенерировать список

**Ожидаемый результат:**  
Красное сообщение: `Не удалось получить ответ. Попробуйте ещё раз.`

- Секреты хранятся **только** в `.env` и не попадают в репозиторий
- Ошибки API обрабатываются без вывода traceback и сырых исключений
- Все сообщения пользователю — короткие и понятные (1 предложение)
